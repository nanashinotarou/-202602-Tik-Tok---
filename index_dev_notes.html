UI ARCHITECTURE

UX設計と技術的意図の解説

1. Cognitive Load Reduction

なぜ「日付」を強調表示（Badge UI）するのか

ユーザーが学習カリキュラムを閲覧する際、最も重要なコンテキストは「いつ（When）」と「なにを（What）」です。 テキストだけでなく、日付を「視覚的なオブジェクト」として独立させることで、スキャナビリティ（流し読みのしやすさ）を向上させ、認知負荷を低減させています。

.date-box { display: flex; flex-direction: column; background: #7986cb; /* Visual Anchor */ }

2. Visual Hierarchy & Unity

色彩設計による統一感

アクションボタンには、ヘッダーやバッジと同一の色相（Hue）を持つグラデーションを採用しました。 彩度（Saturation）を統一することで、重要なアクション要素を目立たせつつも、UI全体としてのノイズを排除し、プロフェッショナルな印象を与えます。

.shortcut-btn { background: linear-gradient(135deg, #5c6bc0, #3949ab); /* Consistent Brand Colors */ }

3. Dual-View Synchronization

カレンダー機能の目的（Purpose）

リスト表示（Linear View）は詳細情報の確認に適していますが、「スケジュールの全体感（Temporal Context）」の把握には不向きです。 カレンダー（Grid View）を並列させることで、ユーザーは「週のどこが忙しいか」「次の休講日はいつか」を直感的に把握できます。

技術的実装（Technical Implementation）

カレンダーとリストを双方向にリンクさせる際、パフォーマンスとUXを最優先しました。

  • Mapping: DOM要素に `data-day` 属性と `id` を付与し、O(1)の計算量で対応関係を特定します。
  • No Auto-Scroll: スクロール位置の強制変更は「User Agency（制御権）」を奪う行為であるため排除し、 純粋な視覚フィードバック（ハイライト）のみを提供しています。
const toggleHighlight = (id, isActive) => { // Pure state toggle, no scroll side-effects const cell = document.querySelector(`.cal-cell[data-day="${id}"]`); cell.classList.toggle('active-sync', isActive); };

4. Accessibility & Navigation

絶対配置（Absolute Positioning）によるナビゲーション

「Design Notes」へのリンクは、コンテンツカード内での `position: absolute` を採用しました。

これは、画面上の固定位置（Fixed）が抱える「メインコンテンツへの被り」リスクを回避しつつ、 カードというコンテナ内での一貫したアクセス性を担保するためです。 インタラクションコストを下げ、開発者向け情報への導線を常に確保しています。
